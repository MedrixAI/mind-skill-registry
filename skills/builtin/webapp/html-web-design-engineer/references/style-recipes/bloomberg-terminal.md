# Recipe: bloomberg-terminal

Adapted from the Garden web-design-engineer recipe family for Mindar single-file HTML web apps.

## Character

Dense operational intelligence. Use for dashboards, monitoring surfaces, financial/scientific metrics, comparison tables, filters, and command-like analysis.

## Layout

- Prioritize high information density with clear scanning lanes.
- Use metric strips, sortable tables, compact charts, side filters, and status panes.
- Keep the first viewport immediately useful; avoid long intro copy.
- Group by workflow: overview, filter, inspect, compare, decide.

## Visual Tokens

- Background: dark neutral or high-contrast light neutral depending on content, but keep contrast accessible.
- Accent: one high-signal color for active state; reserve warning/success colors for semantic meaning.
- Lines: crisp dividers, table rules, compact labels.
- Radius: small, 4-8px.
- Motion: functional only.

## Typography

Use system sans for labels and monospace for numbers, IDs, timestamps, and tabular values.

## Interaction

- Include filters, search, row selection, tabular comparison, chart focus, or scenario toggles.
- Dense controls must wrap or scroll cleanly in 400-800px iframe previews.

## Avoid

- Giant hero sections.
- Decorative empty cards.
- Low-contrast small text.
- Unexplained numbers.
