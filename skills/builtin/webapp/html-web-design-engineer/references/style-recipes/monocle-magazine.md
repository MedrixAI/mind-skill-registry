# Recipe: monocle-magazine

Adapted from the Garden web-design-engineer recipe family for Mindar single-file HTML web apps.

## Character

Editorial, urbane, and curated. Use for narrative explainers, institutional overviews, strategic maps, and learning journeys where the reading rhythm matters.

## Layout

- Use magazine-like hierarchy: strong section openers, pull quotes, side notes, captioned modules, and curated feature blocks.
- Keep the app usable: navigation, tabs, and filters should remain visible and practical.
- Alternate dense evidence sections with quieter interpretive sections.
- Keep long lines under control with readable measure.

## Visual Tokens

- Background: off-white or very light neutral.
- Accent: restrained editorial accent, not neon.
- Borders: thin rules and deliberate dividers.
- Radius: low to medium; avoid bubbly SaaS cards.
- Images: external images are forbidden, so use inline SVG diagrams, type-led layouts, or CSS patterning.

## Typography

Use system serif for feature headings only when appropriate; system sans for UI controls and dense text.

## Interaction

- Good fits: chapter navigation, progressive disclosure, glossary panels, tabs, source citations, and reading progress.
- Interactions should support comprehension.

## Avoid

- Landing-page marketing composition.
- Empty atmosphere.
- Editorial style that hides controls or data.
