# Recipe: tufte-dataink

Adapted from the Garden web-design-engineer recipe family for Mindar single-file HTML web apps.

## Character

Data-ink discipline. Evidence first, decoration last. Use for analytical explorers, source-grounded findings, data stories, and research summaries where clarity matters more than spectacle.

## Layout

- Build around annotations, compact charts, tables, marginal notes, and source callouts.
- Prefer one strong reading column plus side evidence over grid-card overload.
- Make chart takeaways explicit; every metric should have context.
- Use whitespace to separate reasoning layers, not to create empty page height.

## Visual Tokens

- Background: warm or cool off-white.
- Text: dark neutral with high contrast.
- Accent: one muted data accent; secondary accents only for categorical comparison.
- Lines: thin rules and axis-like separators.
- Radius: minimal.

## Typography

Use system serif only for editorial emphasis if it improves reading; otherwise use system sans. No external fonts.

## Interaction

- Filters, toggles, highlighted rows, expandable evidence, and chart annotations are preferred.
- Interaction should reveal more source context, not merely animate.

## Avoid

- Dark dashboards for simple data stories.
- Fake precision or invented metrics.
- Chart junk, loud gradients, and dense color legends without meaning.
