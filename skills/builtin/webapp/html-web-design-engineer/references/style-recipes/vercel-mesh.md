# Recipe: vercel-mesh

Adapted from the Garden web-design-engineer recipe family for Mindar single-file HTML web apps.

## Character

Technical, spatial, and platform-like. Use it for AI systems, developer tools, concept explorers, architecture maps, simulations, and immersive technical demos.

## Layout

- Pair a precise control surface with a visual system map, graph, canvas, or interactive topology.
- Use generous negative space around the main visualization, with compact inspector panels.
- Let thin lines, nodes, coordinate grids, and status readouts carry the technical feeling.
- Keep controls accessible and stable on narrow iframe widths.

## Visual Tokens

- Background: light or dark neutral, but not a one-note dark-blue slab.
- Surfaces: translucent only when readability remains strong; otherwise use solid panels.
- Accent: restrained cyan, blue, green, or monochrome system accent.
- Lines: 1px borders, hairline connectors, subtle grid marks.
- Motion: small state transitions and hover tracing; avoid constant decorative movement.

## Typography

Use clean system sans plus monospace for code, IDs, metrics, coordinates, and telemetry.

## Interaction

- Include hover/focus drilldowns, node selection, inspector details, filters, zoom-like controls, or mode switches when useful.
- Any canvas/SVG scene must have fallback visible content and responsive sizing.

## Avoid

- Empty futuristic decoration with no data meaning.
- Overlapping controls on mobile.
- Blurry glows that reduce legibility.
