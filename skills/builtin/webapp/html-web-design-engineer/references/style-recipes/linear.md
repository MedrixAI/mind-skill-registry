# Recipe: linear

Adapted from the Garden web-design-engineer recipe family for Mindar single-file HTML web apps.

## Character

Focused product software. Quiet, fast, precise, and built for repeated use. The page should feel like a real workspace, not a marketing page.

## Layout

- Prefer app-shell composition: top bar or compact sidebar, primary work area, contextual detail panel.
- Use clear grid alignment, tight grouping, and predictable spacing.
- Keep section headings compact; reserve large type for true page-level state.
- Avoid ornamental hero blocks unless the app genuinely needs an onboarding entry.

## Visual Tokens

- Background: near-white or very pale neutral.
- Surfaces: white or slightly lifted neutral with 1px borders.
- Accent: one crisp product accent for selected states and primary actions.
- Radius: 6-10px. Avoid pill-heavy decoration.
- Shadows: minimal; use borders and spacing first.

## Typography

Use system UI stacks only. Create hierarchy with weight, size, spacing, and density rather than custom fonts.

## Interaction

- Tabs, segmented controls, command buttons, filters, and side panels should feel immediate.
- Selected, hover, focus, disabled, and loading states must be visible.
- Keyboard focus should be obvious and not shift layout.

## Avoid

- Large gradient hero areas.
- Decorative cards inside cards.
- Excessive rounded badges.
- Fake roadmap, fake testimonials, or fake product metrics.
